## Pull Request Process

1. **Before You Start**
   - Check existing issues and PRs
   - Fork and create a descriptive branch name
   - Make focused, single-purpose changes

2. **While Working**
   - Follow the existing code style
   - Keep commits clean and atomic
   - Write clear commit messages
   - Update documentation as needed

3. **Creating Your PR**
   - Use a descriptive title
   - Reference related issues (#issue-number)
   - Explain what changed and why
   - Include any relevant context

4. **Review Process**
   - Maintainers will review your PR
   - May request changes or clarification
   - Once approved, it will be merged
   - You'll be credited as a contributor

## Commit Message Guidelines

Please use clear, concise commit messages:

```
Type: Brief description

- Detail 1
- Detail 2 (if needed)

Fixes #issue-number (if applicable)
```

**Types:**
- `feat:` New feature or content
- `fix:` Bug fix or correction
- `docs:` Documentation updates
- `style:` Formatting or style changes
- `refactor:` Code reorganization
- `test:` Test additions or updates
- `chore:` Maintenance tasks

**Examples:**
```
feat: Add complete Chirala beach itinerary guide
fix: Correct resort contact phone number
docs: Update README with safety information
```
